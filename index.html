2023.04 ~ 현재

  • Codepush 및 Versioning

    • App은 Web과 다르게 Update를 배포하기 위해선 심사라는 과정을 거쳐야 하기 때문에, 문제가 발생한다고 Web처럼 즉시 수정하긴 어려워요.
    • 오류를 빠르게 대응하고 사용함에 불편함 없이 자연스러운 업데이트를 유도 하기 위해 전략을 고민했어요.

  • APP Size 다이어트

    • 기존 App의 용량은 70~80M로 꽤 컸으며, Codepush Bundle 용량 또한 30~40M 정도 되어, Codepush 서버 상태에 따라 느린 속도로 Bundle이 다운되거나, 업데이트가 중단 되어 Rollback User로 변경 되는 등의 서비스에 영향을 미쳤어요.
    • 기존에 사용하고 있던 용량 높은 Design UI 라이브러리를 전부 제거 하고 직접 컴포넌트를 제작하여 대체 했으며, Asset 또한 S3을 통해 내려 받더라도 문제 없는 요소들은 제거해 40MB에 육박했던 Bundle 용량을 10MB 안쪽으로 줄였어요.

  • DeepLink 설계

    • 유저를 원하는 화면으로 이동 시키기 위해 Deeplink를 적극적으로 활용 했어요.
    • 마케팅팀에서 Deeplink를 적극적으로 만들고 사용하기 위해서 Deeplink 주소를 추상화하고 문서로 작성했어요.

  • 상태관리 개선

    • Navigation에 params를 전달하는 방식과 전역스토어를 번갈아 가면서 사용하여 데이터의 흐름의 이해가 어려웠어요.
    • 이를 정리 및 개선하여 단편적으론 Navigation Params를 통해 전달하고 큰 흐름에서는 전역스토어를 사용하도록 개선했어요.

  • Data Fetching 개선

    • React Query로 서버 상태를 관리하고 있지만, 불필요하게 많은 호출과 Background에서 Foreground로 돌아왔을 때 갱신이 되지 않는 문제가 있었어요.
    • 이를 개선 했으며, stale time과 cache time을 조절해 기존 보다 1/3정도 호출 수를 줄였어요.

  • 컴포넌트 제작 및 라이브러리 관리

    • 몇몇 컴포넌트들이 라이브러리에 의존 있었고, 이는 불편한 커스텀과 의존성 문제가 발생 할 수 있었어요.
    • UI 라이브러리들을 제거하고 대부분 직접 구현 했으며, 기존에 있는 UI 컴포넌트들도 새롭게 만들어 사용성을 개선했어요.

    • Web을 간단하게 WebApp형식으로 고객들의 제작 요구가 많아지자, React Native로 간단하게 WebApp을 구축할 수 있는 사내 BoilerPlate를 제작했어요.

  • Project 관리 Notion Template 제작

    • 기존 방식은 Project History나 Project 내용을 찾아 보기가 매우 어려웠어요. 특히 과거에 완료된 프로젝트를 유지 보수 해야 하는 경우엔 더 힘들었어요. 이를 Notion을 활용하면 해결 할 수 있을 거라 생각해 Project 관리 페이지를 Notion Template으로 제작해 사내에서 사용할 수 있게 요청했으며, 새로운 프로젝트부터 도입했어요.

  • 사내 React 세미나 발표

    • 사내 React 세미나를 진행하여, 신규 개발자 또는 BackEnd 개발자에게 React의 작동 방식이나 원리, 최근 FrontEnd의 동향에 대한 내용을 소개했어요.
